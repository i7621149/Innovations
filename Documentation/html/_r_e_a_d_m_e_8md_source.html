1 #Template Project
2 
3 This is a template project, in a similar vein to ngl's BlankNGL. It contains the recommended starting point for any projects that use the shader library.
4 
5 This template includes ShaderLibPro, ShaderVariables, ShaderSet, ShaderPro and Entity, which are all required to use the library and are considered the components of the library.
6 
7 It also includes an NGLScene class and Background, which inherits from Entity. Both of these are recommended for use with my classes, since they are good examples of how to use the library, and the NGLScene will deal with the required ShaderVariables automatically, so that the user does not need to set those up themselves. The Background class is also recommended since it shows how to inherit from Entity, and means that there will always be something in the background rather than the clear color.
8 
9 The user could rewrite or get rid of either of these if they prefer to use the library in their own way.
10 
11 The template also comes with a default shader that shows how it can be written, in this case it is the same as the "new" option in ShaderToy.
12 
13 ##shader text files
14 
15 The shader text files take the following format:
16 ```
17 #comments are made with #s
18 
19 #start with SHADER and shader name. one MAIN is required.
20 SHADER MAIN
21 
22  #specify fragment shader glsl file. This will be combined with sampler2Ds if there are textures and the BaseFragment.glsl file
23  FRAGMENT shaders/DefaultQuadFragment.glsl
24 
25  #specify vertex shader glsl file. This will be combined with the BaseVertex.glsl
26  VERTEX shaders/GeoVertex.glsl
27 
28  #textures can be specified with TEXTURE followed by type (2D) and then the file
29  TEXTURE 2D textures/example.png
30  TEXTURE 2D textures/example2.png
31 
32 #end the shader
33 END MAIN
34 
35 ```
